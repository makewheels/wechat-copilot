# wechat-copilot · 设计

> 自用 AI 微信恋爱军师。本机命令行先跑通，之后再套手机前端。
> 配套：[需求与技术选型.md](需求与技术选型.md)（需求）、[决策记录.md](决策记录.md)（为什么这么选）、[flow.html](flow.html)（流程图）。

## 一句话
把「人设 + 《一秒心动》打法 + 这个人的近期聊天 + 我的意图」塞进**一次** Qwen3-Max 调用，吐出「分析 + 下一句草稿」。没有 agent 框架、没有向量库、没有自主工具——本质就是塞上下文。

## 流程（4 步，一次模型调用）
1. **输入**：选对象 + 她说了啥(`--her`) + 我的意图(`--intent`)
2. **取**：人设/档案 + 书精要 + 这个人最近 N 条原文
3. **🤖 调一次 qwen3-max** → ① 她在想啥 ② 阶段/该干啥 ③ 2-3 条草稿 ④ 风险
4. **给我看**（`--save` 则把这轮原文+意图追加进 data/）

## 代码地图
```
core.py        共享内核：取数据 / 组装上下文 / 调模型（所有逻辑在这）
coach.py       命令行：给某人出下一句   python3 coach.py 王玲 --her "在吗" --intent "约饭"
backtest.py    回测：拿历史真实对话验证军师强不强（质量闸门）
book/一秒心动_精要.md   书蒸馏 = "好的标准"，含油腻过滤
data/<对象>/           按人隔离的存储（gitignore）
  messages.jsonl   原文真相源 {ts,who,type,text}，首次从 zip 自动导入
  intents.jsonl    我的意图/画外音 {ts,note}
  profile.md       慢变档案（可选，手填，有则进 prompt）
docs/                设计文档（本目录）
.env                 DASHSCOPE_API_KEY / QWEN_MODEL=qwen3-max（gitignore）
```

## 数据
首次用某人，从 `~/Downloads/wechat-history/<名字>_<wxid>.zip` 的 messages.json 自动导入并规范化成 `data/<名字>/messages.jsonl`。之后读 jsonl；新对话追加到 jsonl。

## 模型
Qwen3-Max（DashScope，中文母语 + 便宜 + 北京无网络坑）。换模型只改 `.env` 的 `QWEN_MODEL` 或 `core.call_qwen`。

## 相位（每步有闸门）
- **P0** ✅ 内核跑通真实数据（coach.py）
- **P1** ← 当前：回测（backtest.py），比不过当时的我就重调 prompt，别往下做
- **P2** 补 profile 自动维护 / 输出规整（过了 P1 再做）
- **P3** 手机前端（网页两输入框 / 企业微信机器人），只是 core 的一层壳
- **P4** 只有撞到"上下文塞不下"才加记忆压缩（Mem0 或确定性摘要）
