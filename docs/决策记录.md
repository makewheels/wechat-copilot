# 决策记录（ADR）

> 关键技术决策 + 为什么。给之后的 agent / 未来的自己追溯用。均为 2026-06-06 讨论结论。

| # | 决策 | 为什么 |
|---|---|---|
| D1 | **不用 agent 编排框架**（LangChain / OpenClaw / LangGraph） | 任务是"凑上下文调一次模型生成下一句"，没有多步自主循环。框架的编排/工具循环纯负担，单人 debug 不起。 |
| D2 | **暂不用 Mem0 / 向量库 / 记忆层** | 正常微信来回几百条塞得进上下文窗口，不需要"自动压缩/检索"。YAGNI，撞到"塞不下"再加（P4）。 |
| D3 | **模型用 Qwen3-Max**，不上 Opus / OpenRouter | 重中文任务，Qwen 母语、产出地道、便宜、北京调它零网络坑。模型是后置优化，先验证内核；真要比，回测时拿同样 case 横评。 |
| D4 | **本机命令行优先**，不先做服务器/手机 | 先在这台电脑跑通、验证内核值不值。手机/企业微信只是 core 外面一层壳，后置（P3）。 |
| D5 | **教练定位，非代笔**；输出固定四块（她在想啥 / 阶段 / 2-3草稿 / 风险） | 目标是提升用户自己到平均线以上，不是替他把话说满；草稿只是参考，他自己改。 |
| D6 | **原文存 jsonl 真相源；书蒸馏进 system prompt**（非整本 RAG） | "原文很重要"要能逐字回放；书塞整本会上下文过载且噪声大，蒸馏成原则+范例更稳，还能内建"油腻过滤"。 |
| D7 | **回测做质量闸门**（backtest.py） | 这行没有单元测试。用户有 33 个**结局已知**的真实对象(秦艺轩成/张艺薇凉/王玲没约成)=带标签数据集，截断历史让军师出招、对照真实走向，把"凭感觉"变"有证据"。比不过当时的他就重调，别往下做。 |
| D8 | **不挂自动回复微信机器人** | 个人微信自动收发有封号风险；坚持只读 + 人工发（人在回路），零风险。 |
