# 聊天记忆系统 · 计划

> 目标：给「正在聊的人」做记忆，让军师回复更有温度、不像 AI。
> 第一个真实目标：**董丽琼**（`wxid_493f2glh530d22`）。

## 核心决策（拍板了，别再纠结）

| 问题 | 决定 | 为什么 |
|---|---|---|
| 用框架吗（langgraph/mem0）？ | **不用，自己做** | 本质就是「拼上下文→调一次模型→更新记忆」，三个函数，框架只会多一层要调试的东西 |
| 记忆存哪：向量还是文本？ | **文本** `memory.md`，每人一个 | 小、你能一眼看懂、能手改；模型维护，可审计。不是黑盒 |
| 每条消息都更新记忆吗？ | **不**。攒一批，整篇重写 | 增量 merge 会越积越乱、自相矛盾；整篇重写永远干净 |
| 怎么触发更新？ | 每晚 cron 跑一次 / 或手动 | 不需要框架来触发，就是定时跑脚本 |
| 出草稿时怎么调取记忆？ | 整个 `memory.md` 读进 prompt + 最近 ~20 条原文 | 历史再多 token 也恒定；中间旧废话不进 prompt |
| 向量检索要做吗？ | **v1 不做**，v2 可选 | callback「她三个月前提过的事」才需要，先用 memory.md 顶 |

## memory.md 长什么样（四块）

```markdown
## 她是谁（变得慢）
互联网行业，忙、常加班。喜欢旅游但没时间去。

## 关系状态（变得快）
阶段：破冰 | 温度：平，回复短但会回 | 上次进展：—

## 可回扣的细节   ← 温度的真正来源
- （模型从对话里提炼）

## 雷区
- （试过失败的招）
```

初始内容从 `contacts.json` 里她现有的 profile 迁过来，之后由模型维护。

## 出草稿时喂给模型的上下文（固定这几样）

1. 人设 + 打法原则（**精简版**的书，不是现在的全文灌入）
2. 她的 `memory.md`（四块）
3. 最近 ~20 条原文（语气节奏在这，不压）
4. 她最新一条 + 我这轮意图

→ 「消息太多怎么办」在这一步自动解决：永远只喂 `记忆(小) + 近20条`，与总历史长短无关。

## 文件夹分类（项目结构）

现在所有 `.py` 平铺在根目录。**源码先保持平铺**（搬进 src/ 会断 import + relay 的路径，风险大、收益小，单独再说）。先把「类别」定清楚：

```
wechat-copilot/
├── *.py            源码（core/coach/watch/server… 维持平铺）
├── docs/           文档（设计/决策/本计划）← 文档都放这
├── data/<联系人>/  每人的记忆与消息
│   ├── messages.jsonl   原始逐字流
│   ├── memory.md        ← 新增：模型维护的记忆
│   └── tried.jsonl      试过的招+反应（可选，v2）
├── book/           打法知识库（一秒心动精要）
└── relay/          微信镜像转发（独立子系统）
```

新增的源码文件，命名跟现有风格一致，放根目录：
- `memory.py` —— 读/写 memory.md，拼记忆上下文
- `update_memory.py` —— 跑一次：旧记忆 + 新对话 → 整篇重写

# 功能二：聊天记录问答（独立功能）

> 例：「跟房东那事解决了吗？」「李老师上次给我啥建议？」「我和谁聊过装修？」
> 这是在**全部历史**里搜答案，跨人跨话题 —— **这个才需要向量检索**。

## 和功能一的区别（一句话说清「向量还是文本」）

| | 恋爱军师记忆 | 聊天记录问答 |
|---|---|---|
| 存法 | 文本 memory.md，精炼 | 向量索引，全量 |
| 取法 | 整篇读进去 | 按问题检索 top-k 片段 |
| 为什么 | 小、要可改可审计 | 大、要按语义搜 |

## 怎么做（RAG，自己搭，不用框架）

1. **切块**：把消息按「时间段 / 一轮对话」切成小片段，每片带 **谁说的 + 什么时候**（答「老师给了啥建议」必须知道谁是老师、哪天说的）
2. **建索引**：每片 embedding（DashScope 有 text-embedding 接口）。几千条用 numpy 点积就够，几万条再上 faiss
3. **问答**：提问 → embedding → 检索最相关的几片 → 连同问题塞给模型 → 模型**只基于检索到的片段**回答，并附上「来自 X 月 X 日和 XX 的对话」
4. 增量：新消息追加进索引即可

## 步骤

- [ ] **Q1** `index.py`：把指定人(或全部)的 messages 切块 + embedding + 存本地
- [ ] **Q2** `ask.py`：提问 → 检索 → 模型基于片段回答，带出处
- [ ] **Q3** 接进现有入口（命令行 / 飞书 / 网页）

## 验收标准

不是回复率这种指标（这是品味领域，不可量化）。看：
- 草稿你**几乎原样能发** = 对了；每次都大改 = 没抓到你
- 拿几个你印象里「乱开玩笑/没温度」的真实 case 当靶子，改完对照
