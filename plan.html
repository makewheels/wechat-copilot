微信 AI 陪聊助手 · 单 HTML 版

项目代号 wechat-copilot ·读你的微信聊天记录 + 学《一秒心动》→ 生成回复草稿 → 网页里看、复制、去微信发

我要做的
读某个联系人的聊天 → 结合书生成"该怎么回" → 产出一个 HTML,你浏览器打开看草稿、一键复制 → 自己到微信发
为什么这样
WeFlow 只能读不能发,所以"发"这一步由你手动完成(顺带零封号风险)
谁来生成
就是我(Claude),不另搭后端 / 不调外部模型 / 不要 API key

现状盘点(已确认)

  • 本机 Apple M4 / arm64、WeChat 4.1.7 — ✅ 满足 WeFlow 要求
  • WeFlow 还没装,5031 端口未监听 → 第一步要先装
  • WeFlow API 全是读取类(会话/消息/联系人/朋友圈),没有"发消息"接口
  • 已有 ~/workspace/media/wechat-analyzer 是分析产线 → 本项目新建独立目录,只复用它的 weflow_client.py(纯标准库,无依赖)
  • 书已就位:~/Downloads/一秒心动+A5+208.pdf(48MB / 208 页)

形态决策

就一个 HTML 无后端 无服务 无 API key 无构建工具 只做单人 人在回路发送

"读对话 + 结合书生成草稿"由我在会话里做,结果渲染成单个 HTML;HTML 只负责展示草稿 + 复制。先跑通,满意后再谈自动化。

数据流

WeChat(4.1.7) → WeFlow(本机:5031 只读) → fetch.py 拉某人最近对话
《一秒心动》PDF → 抽文本 → 风格/话术要点(我提炼)
      → 我读「对话 + 书要点」生成 1~3 条候选回复
      → 渲染成 reply.html(对话回顾 + 候选草稿 + 每条「复制」按钮)
      → 浏览器打开 reply.html → 复制 → 去微信发

执行步骤

1

安装并启动 WeFlow (你在界面操作,我指导)

  • GitHub Releases 下 arm64 的 .dmg → 拖进 /Applications
  • 首启被拦:系统设置 → 隐私与安全性 → 仍要打开;读库可能需在完全磁盘访问权限里勾选 WeFlow
  • 微信保持登录 → WeFlow 加载数据 → 设置 → API 服务 → 启动服务(默认端口 5031),记下 access_token
  • 验证:curl http://127.0.0.1:5031/api/v1/health 正常
2

提炼书要点

抽出 PDF 文本,我读后提炼成精简要点(核心话术原则 + 语气风格 + 范例句式),生成时参考——不每次塞整本 208 页。
3

新建项目 ~/workspace/tools/wechat-copilot/

  • 复用 weflow_client.py(从 analyzer 拷一份)
  • fetch.py:给联系人关键词 + 条数,解析 wxid、拉最近对话、清洗成 时间 / 谁 / 文本
  • .env + .gitignore(token、PDF、output、reply.html 不入库)
4

陪聊循环

你说"帮我回 XX" → 我跑 fetch.py 取对话 → 结合书生成 1~3 条候选 → 渲染 reply.html → 你浏览器打开、复制满意的一条 → 去微信发。

前提与风险

注意

  • WeFlow 读库权限:Mac 上可能需"完全磁盘访问权限"且微信在登录态;拉不到数据按此排查
  • 封号风险:全程不自动发微信(人在回路),风险极低
  • 隐私:聊天记录、书、草稿全程本机处理,token 走 .env

怎么算跑通

  • WeFlow API 健康、能列会话
  • fetch.py 能拿到某人正确的最近对话
  • 生成的 reply.html:草稿贴合《一秒心动》风格、契合当前上下文,复制按钮可用
这就是 reply.html 大概长什么样的预览 —— 真正的草稿页会在这个壳子里放候选回复 + 复制按钮
